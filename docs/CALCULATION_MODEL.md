# Расчетная модель

## Общий принцип

Приложение не хранит вручную собранную математическую модель. Вместо этого оно извлекает структуру из `Engineering Efficiency Measurement.xlsx`, а затем считает результаты в Python.

Это означает:

- факторы `PLAN` берутся из workbook;
- qualitative-to-numeric mapping берется из workbook;
- показатели `PLAN` и `FACT` берутся из workbook;
- коэффициенты и веса берутся из workbook;
- линейные функции нормализации берутся из workbook.

## PLAN

### Источники

- лист `input I`
- лист `engineering efficiency PLAN`

### Этап 1. Выбор qualitative-оценки

Для каждого фактора пользователь выбирает qualitative-значение. Например:

- `very low`
- `low`
- `moderate`
- `high`
- `very high`

или соответствующий текстовый вариант, который реально указан в workbook.

### Этап 2. Переход к числовому значению

Каждому qualitative-уровню соответствует числовое значение. Приложение подставляет его из таблицы workbook.

### Этап 3. Расчет raw-value показателя

Для каждого показателя `PLAN` workbook задает:

- intercept;
- коэффициенты по факторам;
- границы `worst` и `best`;
- вес.

Формула имеет вид:

```text
raw_value = intercept + sum(coefficient_i * factor_value_i)
```

### Этап 4. Нормализация

Далее raw-value преобразуется к шкале `[0..1]` по той же piecewise-логике, что и в Excel.

Если показатель “чем меньше, тем лучше”:

- значение хуже worst дает `0`;
- значение лучше best дает `1`;
- между ними используется линейная формула workbook.

Если показатель “чем больше, тем лучше”, логика зеркальная.

### Этап 5. Взвешивание

```text
weighted_score = normalized_score * weight
```

### Этап 6. Итог

Итоговый `PLAN` равен сумме weighted scores всех показателей. Это эквивалент ячейки `Y16`.

## FACT

### Источник

- лист `engineering efficiency FACT`

### Этап 1. Ввод факта

Пользователь вводит фактические значения по показателям `y1..y10`.

### Этап 2. Нормализация

Для каждого FACT-показателя workbook задает:

- худшую границу;
- лучшую границу;
- линейную формулу между ними;
- вес.

Приложение повторяет эту же логику:

- вне рабочей границы значение срезается к `0` или `1`;
- внутри диапазона используется линейная формула workbook.

### Этап 3. Взвешивание и итог

```text
weighted_score = normalized_score * weight
```

Итоговый `FACT` равен сумме всех weighted scores. Это эквивалент ячейки `N15`.

## Почему используются slope и intercept

Excel использует линейные формулы с коэффициентами вида:

```text
score = slope * value + intercept
```

Чтобы совпадать с workbook, приложение извлекает:

- `score_slope`
- `score_intercept`

и применяет именно их, а не приближенную интерполяцию.

- расчетную логику.

## Ограничения

- Текущий парсер рассчитан на текущую структуру workbook.
- Если поменяются названия листов или формат формул, парсер нужно адаптировать.
- Сейчас модель загружается из workbook на старте приложения, а не кэшируется отдельно в JSON.
